Detailed Description

A single object of this class exists. It encapsulates the functionality of the Arduino standard library LiquidCrystal and reduces it to the printf functions needed for this application of the display. Furthermore the print functions implement all needed task synchronization: The display is shared by several tasks, which will all write their specific information into the display.

The constructor. It is not accessible from outside as there is only one instance of this class. This single object is plublic, not the constructor to make it.

Class construtor. Only a single instance of the class must exist in an application, therefore this constructor must never be used.

Remarks:
The configuration of the underlaying class LiquidCrystal, particularly the supported pins, is done hardcoded in the implementation of this constructor.

void dpy_display_t::printGreeting ( )

Print a greeting after reset.

Print a standard greeting text. This function should be called immediately after reset.
The function must not be called at run time, when concurrent tasks try to access the display. This function will not acquire the mutex for safe access of the display.

To be called once first and typically only once at all: The invariant background parts of the layout are printed. This print may overwrite any of the others.

void dpy_display_t::printAdcInput ( int8_t  idxInput)

Formatted printing of ADC input selection.

Print the currently selected ADC input (or multiplexer input respectively).
The function can be called only at run time from an RTuinOS task other than the idle task. (It acquires the mutex for safe access of the display, which is forbidden for the idle task.)

Parameters:
idxInputThe index of the selected ADC input. The range is [-1..15]. (A value of -1 can be passed to indicate that none of the external multiplexer inputs is currently chosen.) Exceeding the range will lead to a runtime error!
